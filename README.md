##  Overview

The **Integrated Service Request, Inventory, and Client Payment Transaction System** is a web-based solution developed for **ECL-Micro Network & Data Solutions**.  
It replaces **manual, paper-based, and face-to-face other workflows** with a **centralized digital platform** that improves efficiency, accuracy, and record management.

This repository represents the **initial system design and prototype** for academic and organizational planning purposes.

---

##  Objectives

### General Objective
To develop an integrated web-based system that automates service requests, inventory tracking, and client payment transactions.

### Specific Objectives
- Enable clients to submit and track service requests  
- Monitor inventory items and stock levels  
- Record and manage client payment transactions  
- Generate system reports for decision-making  
- Implement role-based access control  

---

##  System Scope

This system is intended **exclusively for internal use**.

### Included Features
- Service request submission and monitoring  
- Inventory management and stock tracking  
- Payment transaction recording  
- Basic reporting tools  
- Role-based access (Admin, Staff, Client)  

---

##  Limitations

- Single-branch support only  
- No mobile application  
- No third-party API integrations  
- Limited analytics and dashboards  
- No enterprise-level scalability testing  

---


##  Key Features

- Web-based service request system  
- Real-time inventory tracking  
- Centralized payment monitoring  
- Admin dashboard summary  
- Secure role-based access  
- Searchable service records
